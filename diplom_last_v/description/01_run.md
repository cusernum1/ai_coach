# run.py — Файл для запуска проекта

## За что отвечает файл

Это самый маленький файл в проекте — всего 4 рабочие строки. Его единственная задача — дать пользователю удобный способ запустить программу командой `python run.py`, не запоминая более сложную команду `python -m app.main`.

## Полный код с объяснениями

```python
# ============================================================
# run.py — шорткат для запуска: python run.py
# ============================================================
# Эквивалент `python -m app.main`. Добавлен для удобства:
# студенту/проверяющему не нужно помнить флаг `-m`.
# ============================================================
```
**Строки 1-7:** Комментарий-шапка. Объясняет назначение файла. В Python комментарии начинаются с `#` и не выполняются.

---

```python
from app.main import main
```
**Строка 8:** Импортируем функцию `main` из файла `app/main.py`. Это как сказать: «возьми функцию `main` оттуда». После этой строки функция `main` доступна в нашем файле.

---

```python
if __name__ == "__main__":
```
**Строка 10:** Специальная проверка Python. Переменная `__name__` равна `"__main__"` только когда файл **запускается напрямую** (`python run.py`). Если файл просто импортируется другим модулем, `__name__` будет равно `"run"` и код внутри `if` не выполнится. Это защита от случайного запуска при импорте.

---

```python
    import asyncio
```
**Строка 11:** Импортируем библиотеку `asyncio` — стандартный модуль Python для асинхронного программирования. Асинхронность нужна потому, что наш бот, веб-сервер и планировщик работают **одновременно** в одном процессе.

---

```python
    asyncio.run(main())
```
**Строка 12:** Запускаем нашу функцию `main()` через `asyncio.run()`. Это обязательно для асинхронных функций (помеченных `async def`) — они не могут запуститься сами по себе через обычный вызов. `asyncio.run()` создаёт «event loop» (цикл событий) и передаёт ему нашу функцию.

## Как работает

```
python run.py
    ↓
__name__ == "__main__" → True
    ↓
asyncio.run(main())
    ↓
запускается app/main.py::main()
    ↓
стартуют бот + веб-сервер + планировщик
```

## Ключевые термины для защиты

- **`asyncio`** — библиотека Python для запуска нескольких задач «одновременно» (на самом деле по очереди, но очень быстро переключаясь).
- **`asyncio.run()`** — запускает асинхронную функцию. Без этого `async def main()` не запустится.
- **`if __name__ == "__main__"`** — стандартная защита: код выполняется только при прямом запуске файла.
