# __init__.py файлы — что это и зачем они нужны

## Что такое __init__.py вообще

Представь, что папка с кодом — это коробка. Чтобы Python понял, что это не просто папка с файлами, а **пакет** (модуль, который можно импортировать), в ней должен лежать файл `__init__.py`. Это как «паспорт» папки: видишь `__init__.py` — значит, это пакет Python.

Без `__init__.py` написать `from app.db import get_session` не получится — Python просто не найдёт папку.

---

## Все __init__.py в проекте

### `app/__init__.py`
```python
# ============================================================
# app — корень пакета AI-Coach v3
# ============================================================
```
**Что делает:** ничего, кроме того, что превращает папку `app/` в пакет Python.  
Комментарий служит как заголовок — чтобы сразу было понятно, что это корень проекта.

---

### `app/bot/__init__.py`
```python
# ============================================================
# app/bot — Telegram-бот (aiogram 3)
# ============================================================
```
**Что делает:** объявляет `app/bot/` пакетом. Именно поэтому можно писать `from app.bot.main import build_bot`.

### `app/bot/handlers/__init__.py`
```python
from aiogram import Router
from . import agent_chat, athlete, coach, common, payments, poll, strava, training_log

def setup_routers() -> Router:
    """Собирает корневой router со всеми дочерними."""
    root = Router(name="root")
    root.include_router(common.router)
    root.include_router(coach.router)
    root.include_router(athlete.router)
    root.include_router(poll.router)
    root.include_router(training_log.router)
    root.include_router(payments.router)
    root.include_router(strava.router)
    root.include_router(agent_chat.router)  # ПОСЛЕДНИМ — ловит любой текст
    return root
```
**Что делает:** этот `__init__.py` не пустой — он выполняет важную роль «сборщика».

- `from . import agent_chat, athlete, ...` — импортирует все файлы-обработчики из этой же папки (точка `.` значит «текущая папка»).
- `setup_routers()` — собирает все роутеры в один главный роутер.
- **Порядок важен!** `agent_chat` подключается последним, потому что он ловит **любой текст**. Если поставить его первым, он перехватит все команды (/start, /plan и т.д.) раньше, чем другие обработчики.

### `app/core/__init__.py`
```python
# ============================================================
# app/core — Домен: LLM-агент, метрики, промпты
# ============================================================
# В этом слое хранятся знания о предметной области
# (спортивный тренер) — независимо от транспорта (бот/веб).
# ============================================================
```
**Что делает:** просто объявляет пакет. Комментарий объясняет концепцию «domain layer» — эти модули не знают ни про Telegram, ни про веб, только про спорт и ИИ.

### `app/db/__init__.py`
```python
from app.db.database import Base, get_session, init_db, engine  # noqa: F401
from app.db import models  # noqa: F401
from app.db import repo    # noqa: F401
```
**Что делает:** «переэкспортирует» самое нужное из папки `db/`.

- `from app.db.database import Base, get_session, init_db, engine` — теперь в любом другом файле можно писать коротко `from app.db import get_session` вместо длинного `from app.db.database import get_session`.
- `# noqa: F401` — подсказка линтеру «не ругайся на "импортировано, но не используется" — это сделано намеренно для реэкспорта».

## Итого: зачем нужны __init__.py

| Файл | Роль |
|------|------|
| `app/__init__.py` | Корень всего пакета |
| `app/bot/__init__.py` | Пакет бота |
| `app/bot/handlers/__init__.py` | **Активный** — собирает все роутеры |
| `app/core/__init__.py` | Пакет бизнес-логики |
| `app/db/__init__.py` | **Активный** — реэкспортирует get_session, init_db |
| `app/integrations/__init__.py` | Пакет интеграций |
| `app/scheduler/__init__.py` | Пакет планировщика |
| `app/webapp/__init__.py` | Пакет веб-приложения |

**Простое правило:** если папка содержит Python-код, который нужно импортировать — в ней должен быть `__init__.py`. Большинство из них пустые (только комментарий-заголовок), но `handlers/__init__.py` и `db/__init__.py` выполняют реальную работу.
