---

```python
    expire_on_commit=False,
```
**Важная настройка!** По умолчанию после `commit()` все атрибуты ORM-объектов «протухают» — SQLAlchemy помечает их устаревшими. При следующем обращении (`user.name`) автоматически делался бы SQL-запрос для обновления. Но наши handlers обращаются к объектам **после** закрытия сессии — и тогда запрос невозможен. `expire_on_commit=False` отключает это протухание.

---

```python
    autoflush=False,
```
Не сбрасывать изменения в БД автоматически перед каждым SELECT. Мы вызываем `flush()` вручную в репозиториях, чтобы лучше контролировать момент записи.

---

```python
    try:
```
Начало блока обработки ошибок. Всё что внутри — выполняется в одной транзакции.

---

```python
        yield session
```
**Ключевая строка.** `yield` превращает функцию в генератор. Здесь выполнение **приостанавливается** и управление передаётся в блок `async with`. Код внутри `async with get_session() as s:` получает `session` как переменную `s`.

После выхода из блока `async with` — выполнение **возобновляется** со следующей строки (после `yield`).

---

```python
        await session.commit()
```
Если блок `async with` завершился **без ошибки** — сохраняем все изменения в БД. `commit()` делает `COMMIT` в SQL — все INSERT/UPDATE/DELETE становятся постоянными.

---

```python
    except Exception:
```
Если внутри блока `async with` возникло **любое исключение** — выполнение прыгает сюда.

---

```python
        await session.rollback()
```
Отменяем **все изменения** текущей транзакции. Как будто ничего не было. `ROLLBACK` в SQL.

---

```python
        raise
```
Перебрасываем исключение дальше — не «проглатываем» ошибку. Вызывающий код тоже увидит ошибку.

---

```python
    finally:
```
Этот блок выполняется **всегда** — и при успехе, и при ошибке.

---

```python
        await session.close()
```
Закрываем сессию — возвращаем соединение в пул. Без этого соединения накапливались бы и пул исчерпался.

---

```python
    from app.db import models  # noqa: F401
```
Импортируем модуль `models` внутри функции. Это **обязательно** — при импорте SQLAlchemy регистрирует все классы в `Base.metadata`. Без этой строки `create_all` «не знал» бы о наших таблицах и ничего не создал.

`# noqa: F401` — подсказка линтеру не ругаться на «импортировано, но не используется» — нам нужен сам факт импорта (побочный эффект).

---

```python
    async with engine.begin() as conn:
```
Открываем соединение в режиме транзакции. `engine.begin()` — контекстный менеджер, автоматически делает `commit` при выходе или `rollback` при ошибке.

---

```python
        await conn.run_sync(Base.metadata.create_all)
```
Выполняем синхронную функцию `create_all` в асинхронном контексте через `run_sync`.

`Base.metadata.create_all` — создаёт **все таблицы** описанные во всех классах унаследованных от `Base`. Если таблица уже существует — пропускает её (не пересоздаёт).

---

```python
    logger.info("DB schema ensured (Base.metadata.create_all)")
```
Пишем в лог подтверждение что таблицы созданы/проверены.
