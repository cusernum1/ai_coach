Интеллектуальная система для автоматизации работы спортивного тренера на базе LLM-агента с поддержкой function-calling, мониторингом тренировочной нагрузки и персональной аналитикой.

**Дипломная работа** · Python · Streamlit · OpenRouter / Groq · SQLite · Pydantic

---

## 📖 Описание

Система представляет собой AI-ассистента для спортивных тренеров и спортсменов, который автоматизирует рутинные задачи: составление тренировочных планов, анализ прогресса, оценку восстановления, рекомендации по питанию. В основе — LLM-агент, реализующий паттерн **ReAct** (Reasoning + Acting) с использованием function-calling для выбора подходящего инструмента под каждый запрос.

### Ключевые особенности

- 🤖 **Умный агент** с function-calling — автоматически выбирает нужный инструмент
- 🧠 **Память контекста** — агент помнит предыдущие сообщения в чате
- 📊 **ACWR-метрика** — Acute:Chronic Workload Ratio для предотвращения травм
- 📈 **Wellness Score** — интегральный показатель самочувствия (0–100)
- 🏋️ **Training Load** — мониторинг тренировочной нагрузки по RPE
- 📓 **Журнал тренировок** — отслеживание выполнения плана
- 📕 **Экспорт в PDF/TXT** с поддержкой кириллицы
- ✅ **Оценка качества ответов** агента по доменным ключевым словам
- 🧪 **Полное покрытие тестами** (pytest + mocking)

## 🏗️ Архитектура

Проект построен по принципу **слоистой архитектуры** с чётким разделением ответственности:

```
┌─────────────────────────────────────────────┐
│            Streamlit UI (app.py)            │  ← Presentation Layer
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│       Service Layer (services.py)           │  ← Business Logic
│        CoachService — фасад логики          │
└─┬──────────┬──────────┬──────────┬──────────┘
  │          │          │          │
  ▼          ▼          ▼          ▼
┌────┐  ┌────────┐  ┌────────┐  ┌──────────┐
│LLM │  │Database│  │Metrics │  │ Models   │
│Agent│  │(SQLite)│  │(ACWR,..│  │(Pydantic)│
└────┘  └────────┘  └────────┘  └──────────┘
  │
  ▼
┌──────────────────────────────────┐
│  OpenRouter (gpt-oss-120b:free)  │
│            или Groq              │
└──────────────────────────────────┘
```

### Паттерны и принципы

- **ReAct** — цикл рассуждение → действие → наблюдение
- **Facade** — `CoachService` скрывает сложность взаимодействия модулей
- **Repository** — `database.py` изолирует операции с БД
- **Dependency Injection** — конфигурация через `config.py`
- **Fail-fast validation** — Pydantic валидирует данные на входе

## 🚀 Быстрый старт

### Требования

- Python 3.10+
- API-ключ OpenRouter ([получить бесплатно](https://openrouter.ai/keys)) или Groq ([получить бесплатно](https://console.groq.com/keys))

### Установка

```bash
# 1. Клонируйте репозиторий
git clone <repository-url>
cd ai-coach

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/macOS
# или venv\Scripts\activate  # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Настройте API-ключи
cp .env.example .env
# Откройте .env и вставьте ваш OPENROUTER_API_KEY

# 5. Скачайте шрифты для PDF (один раз)
python download_fonts.py

# 6. Запустите приложение
streamlit run app.py
```

Приложение откроется по адресу `http://localhost:8501`.

## 🧠 Инструменты агента

Агент использует **5 инструментов** (function calling) — LLM сам выбирает, какой применить:

| Инструмент | Триггеры | Возвращает |
|------------|----------|------------|
| `generate_training_plan` | «составь план», «программа тренировок» | Детальный план на 1–4 недели |
| `analyze_progress` | «проанализируй результаты», «оцени прогресс» | Анализ с выделением сильных/слабых сторон |
| `recovery_recommendation` | «усталость», «восстановление», «болит» | Рекомендации на 24/48 ч + красные флаги |
| `nutrition_recommendation` | «питание», «рацион», «КБЖУ» | КБЖУ + меню + гидратация |
| `analyze_workload` | «анализ нагрузки», «журнал тренировок» | Оценка по RPE и выполнению плана |

## 📊 Уникальные метрики

### 1. Wellness Score (0–100)

Интегральный показатель самочувствия:

```
wellness = (10 - fatigue) × 5 + sleep × 5
```

| Диапазон | Метка | Рекомендация |
|----------|-------|--------------|
| 70–100 | 🟢 Отличное | Полная нагрузка |
| 50–69 | 🟡 Хорошее | Нормальная нагрузка |
| 30–49 | 🟠 Среднее | Снизить интенсивность |
| 0–29 | 🔴 Плохое | Восстановление |

### 2. ACWR (Acute:Chronic Workload Ratio)

Ключевая метрика спортивной науки для **предотвращения травм**:

```
ACWR = средняя нагрузка за 7 дней / средняя нагрузка за 28 дней
```

| ACWR | Зона | Риск |
|------|------|------|
| < 0.8 | Недотренированность | Потеря формы |
| 0.8–1.3 | 🟢 **Оптимальная (sweet spot)** | Минимальный |
| 1.3–1.5 | 🟡 Зона внимания | Средний |
| > 1.5 | 🔴 Опасная | Высокий риск травмы |

### 3. Training Monotony

Показатель однообразия нагрузки (по формуле Foster):

```
monotony = средний RPE / стандартное отклонение RPE
```

Высокая монотонность → риск перетренированности.

### 4. Response Quality

Автоматическая оценка качества ответов агента по присутствию доменных ключевых слов. Позволяет в реальном времени следить за качеством LLM.

## 🔬 Научная основа

Проект опирается на современные исследования в области спортивной науки:

1. **ACWR** — Gabbett T. (2016) *The training—injury prevention paradox*
2. **RPE scale** — Borg G. (1982) *Psychophysical bases of perceived exertion*
3. **Training Monotony** — Foster C. (1998) *Monitoring training in athletes*
4. **Function calling agents** — Yao et al. (2022) *ReAct: Synergizing Reasoning and Acting*
5. **Periodization** — Bompa T., Buzzichelli C. (2019) *Periodization: Theory and Methodology*

### Ключевые тезисы защиты

- **Проблема**: Тренеры тратят 30–60 мин на составление плана одного спортсмена
- **Решение**: ИИ-агент генерирует план за 15–30 секунд с учётом профиля
- **Новизна**: Интеграция классических спортивных метрик (ACWR, Wellness) с LLM
- **Архитектура**: Слоистая, с разделением UI → Service → Data → AI
- **Валидация**: Pydantic + 40+ тестов + мок LLM для CI
